Core Components of Kubernetes

Cluster

A cluster is a set of nodes that run containerized applications. It consists of a control plane and worker nodes that work together to run your applications reliably.

Control Plane

The control plane manages the state of the cluster. It includes the API Server, Scheduler, Controller Manager, and a key-value store (etcd) to persist cluster state.

Worker Nodes

Worker nodes run the containerized applications. Each node typically runs a container runtime (e.g., Docker), the kubelet (agent), and kube-proxy (networking component).
