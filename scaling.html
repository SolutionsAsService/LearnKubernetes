Scaling Your Application

Understand how to scale your application by adjusting the number of replicas in your Deployment configuration.

By increasing the number of replicas, you can improve application availability and load-handling capacity. For example, changing the replicas field from 1 to 3 will instruct Kubernetes to run three instances of your application. 


apiVersion: apps/v1
kind: Deployment
metadata:
  name: scalable-app
  labels:
    app: scalable-app
spec:
  replicas: 3   # Increase the number of replicas to 3
  selector:
    matchLabels:
      app: scalable-app
  template:
    metadata:
      labels:
        app: scalable-app
    spec:
      containers:
      - name: scalable-app-container
        image: nginx:latest
        ports:
        - containerPort: 80

After updating the YAML file, apply the changes using kubectl apply -f scaling.yaml (or your chosen filename), and Kubernetes will manage the scaling process automatically.
